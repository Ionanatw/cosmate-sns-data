## 4. 除錯與教訓

### 除錯 1：v2.0 腳本 PUT Dashboard 清除 Tab 結構（上次 session 留下的 bug，本次修復）
- **問題**：Dashboard Tab 全消失，19 張 cards 扁平堆疊
- **根因**：`PUT /api/dashboard/{id}` 未帶 `tabs` 參數，未在 dashcards 帶 `dashboard_tab_id`
- **解法**：PUT body 同時帶 `tabs`（含 temp 負數 ID）+ `dashcards`（含 `dashboard_tab_id` 對應 temp Tab ID），Metabase 會自動 resolve 為真實 Tab ID
- **教訓**：未來任何 Metabase Dashboard PUT 一定要檢查：① tabs 是否帶入 ② 每個 dashcard 是否帶 dashboard_tab_id
- **🔁 寫進不二錯？**：是（分類：Metabase API；描述：PUT /api/dashboard 若未帶 tabs + dashboard_tab_id 會清除所有 Tab 結構）

### 除錯 2：cosmate_production 無 chat_logs 表
- **問題**：v3 架構規劃頁的漏斗定義引用了 chat_logs，但實際不存在
- **根因**：架構規劃頁是假設性設計，未驗證實際 DB schema
- **解法**：用 permanent_chatrooms 的內建欄位（message_count_member_1/2、last_message_at、dispatching_topic_id）重新定義漏斗各層
- **🔁 寫進不二錯？**：否（已在 metabase_build_v30.py description 欄位備註）
